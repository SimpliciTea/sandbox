Project: Javascript/jQuery Etchasketch

SQUARES PER SIDE:

Another fun one to implement! This was an exercise in programmatically writing HTML using jQuery, and binding event handlers to said HTML.

Features:

  • Adds 20% opacity to elements on each mouseover
  • Customizable resolution! Giv'er a shot.

Challenges:

  • Not all resolutions divide 400x400 cleanly. If you enter a grid dimension that is not a factor of 400, you end up with a 1 pixel column of whitespace on the right. Fixed by setting box-sizing: border-box on all page elements in pages CSS
  • The whole project was a bit of a flex for me -- planning out how to deal with styles and bind events to elements that aren't in the HTML until the page loads was a bit frustrating, but very satisfying upon completion.

Processing.JS exercise: Fishtank

This is my implementation of a KhanAcademy programming project, which can be found here.

It is an example of an HTML5 canvas element being drawn on using the Processing.JS programming language. The advent of the Canvas element in HTML5 brought graphics natively to the web, and so Processing was migrated to be used as a JavaScript library. It's all pretty straightforward once you get the hang of it!

Features:

  • Left-Click to draw randomly generated fish! (size of body, eyes, tail, color of body, tail)
  • Right-Click to clear and redraw a fresh canvas with new seawead
  • Randomly generated seawead using Bezier curves originating from base of canvas frame (try refreshing!)

Challenges:

  • Does not render on htmlpreview/rawgit! Due to cross-origin security in browsers. Check the link to Khan Academy or run on a localhost (sorry!!) fixed by inlining the Processing script
  • Figuring out how Bezier curves and their control points work!
  • Selecting appropriate ranges for Bezier fn points to draw convincing seaweed
  • Figuring out how to embed the canvas in my own page with Processing.JS
  • ...which required setting up my first local webserver using Python!!

Project: Google Homepage HTML/CSS recreation (static)

Click to see me live via rawgit!

Features

  • Clean, semantic HTML/CSS
  • Very close to spacing/styling used by Google
  • Scales well for desktop, laptop, tablet sized screens

Challenges

  • Finding the correct spritesheet for the top-nav icons was tricky
  • Setting the spritesheet in the correct, sizable, positionable containers, then doing so with clean spacing

Afterthoughts

This was a pretty fun little project! It's surprising how much code is sitting behind Google's homepage. What looks like a relatively simple page is hundreds upon hundreds of lines of source code to dig through. Thank god for in-browser developer tools! There are a billion deprecated CSS rules on every element and class it seems, so it was pretty wonderful to be able to look through computed styles of an element rather easily. The toplinks and using my first spritesheet for reactive icons was unexpectedly challenging. Once the top nav fell into place, the rest of the page was pretty well straightforward to build.
